     2. Scope-aware proving in /j

     Instead of "prove 20% batch" vs "prove all", the scope is determined by changed files:

     1. /j collects changed file paths (from the scope the user chose: uncommitted, unpushed, or both)
     2. Match changed paths against paths: frontmatter of each rule file in .proofs/config.json
     3. For matching rule files: prove.sh select-all (prove all tags — these are actively relevant)
     4. For non-matching rule files: skip (they're unrelated to current changes)
     5. CLAUDE.md and philosophy.md (no paths, global): always included

     This means /j only proves documentation covering the areas you actually changed. Can extend to test coverage: rules could also reference test files, and /j validates that tests exist for proved entries.

     prove.sh change

     Add prove.sh select-matching <changed-files...> subcommand:
     - Reads each doc from config.json, parses paths: frontmatter
     - Matches changed files against each rule's globs
     - For matching rules: outputs all tags (like select-all)
     - For non-matching: skips

     This keeps the scope logic in the script (not in /j instructions) so any command can use it.
