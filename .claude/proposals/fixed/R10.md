## Findings

### F1 — kill_process_tree_sync Windows BFS is O(N × descendants) with PID-race
- **Category**: bug / perf
- **Confidence**: high
- **Diff**: medium (~50 LOC)
- **Location**: process.rs:33-86
- **Today**: BFS rescans full `Vec<(pid, ppid)>` for every parent pop. Snapshot taken once at entry, BFS continues against stale list — PID recycle window means `TerminateProcess` may hit unrelated newly-created PID.
- **Proposed**: Build `HashMap<ppid, Vec<pid>>` once (O(N)). For PID-recycle resistance, capture creation_time via GetProcessTimes; only terminate when times match. Better: switch to `sysinfo` crate (already a dep — used by metrics.rs) which provides `parent()` + `start_time()` cross-platform.

### F2 — kill_session_holder_sync / kill_orphan_sessions_sync shell out to wmic/pgrep
- **Category**: perf / portability
- **Confidence**: high
- **Diff**: medium (~80 LOC)
- **Location**: process.rs:125-254 and :297-511
- **Today**: WMIC is **deprecated and removed from Windows 11 24H2 default install**. `kill_orphan_sessions` runs at app startup — missing wmic blocks init for full 10s timeout (:286-294).
- **Proposed**: Use `sysinfo::System::processes()` exclusively. One snapshot serves all lookups. Drops `creation_flags(0x08000000)`. Latency: O(seconds) → O(20ms).

### F3 — kill_orphan_sessions_sync `pgrep -x` matches our own job-object children
- **Category**: bug
- **Confidence**: medium
- **Diff**: small
- **Location**: process.rs:454-471
- **Today**: `pgrep -x <exe-name>` filters my_pid but not job-object descendants. With Linux `PR_SET_CHILD_SUBREAPER` (lib.rs:79), adopted children of crashed CLIs reparent to us — `pgrep -x` returns those.
- **Proposed**: After F2 sysinfo conversion, filter by exe name AND ancestry: exclude any matching PID whose ancestry contains my_pid.

### F4 — Linux send_notification wait_for_action blocks spawn_blocking thread
- **Category**: bug / resource leak
- **Confidence**: high
- **Diff**: small (~15 LOC)
- **Location**: process.rs:557-580
- **Today**: Tauri command awaits `JoinHandle` whose worker calls `wait_for_action` — blocks 5-30s on KDE/GNOME, **infinite on dunst with `expire_time = 0`**. Each pending notification pins one spawn_blocking worker (cap 512). 100 ignored = 100 pinned.
- **Proposed**: After `.show()`, detach handle into `std::thread::spawn`. Tauri command returns immediately; detached thread runs `wait_for_action`. Match Windows path (already fire-and-forget).

### F5 — parse_porcelain `_ => 'M'` masks unknown XY codes silently
- **Category**: bug
- **Confidence**: medium
- **Diff**: trivial
- **Location**: git.rs:92-104
- **Today**: Falls back to 'M' for any unmatched (X,Y). Unmatched: `T` (typechange), `U`/UU/AA/DD/AU/UA/UD/DU (unmerged conflicts). Conflict files showing as 'M' is a UX bug.
- **Proposed**: Add `('U',_)|(_,'U') => 'U'` and `('T',_)|(_,'T') => 'T'`. Add `log::debug!` for final `_` arm.

### F6 — parse_porcelain breaks whole parse on first malformed entry

- **Proposed**: Replace `break` with `i += 1; continue;` and warn at `log::debug!`.

### F10 — INDEX_CACHE in path_resolve.rs violates "NO caching" rule
- **Category**: rule violation / design
- **Confidence**: high
- **Diff**: large (~80 LOC)
- **Location**: path_resolve.rs:9-161
- **Today**: `LazyLock<Mutex<HashMap<String, Arc<CwdIndex>>>>` keyed by cwd, 60s TTL, 16-entry eviction. CLAUDE.md: "Do not use caching, focus on performance optimisation instead."
- **Proposed**: (1) Remove cache outright — WalkBuilder over 100k files takes 50-300ms on modern SSD; resolve_paths is per-batch from terminal link detection. (2) If walk cost too high: switch to `notify` file-watcher rooted at cwd, no TTL, always fresh.

### F13 — SessionManager 3-Mutex coordination invites torn reads and deadlock-by-construction
- **Category**: design / concurrency
- **Confidence**: high
- **Diff**: medium (~120 LOC, isolated)
- **Location**: session/mod.rs:9-88
- **Today**: Three mutexes (sessions, tab_order, active_tab). `set_active` holds sessions then takes active_tab. **`list_sessions` (:80-87) locks sessions then tab_order — torn read possible**: between locks, `add_session` (which inserts into sessions first) means `list_sessions` returns a session whose id isn't in `order` yet → silently dropped by `filter_map`.
- **Proposed**: Single `RwLock<SessionState>`. All operations take `state.write()` (or `.read()` for list_sessions). Five lock sites collapse to one.

### F14 — persist_sessions_json blob-writes frontend JSON with no schema version, no validation, no atomic write
- **Category**: bug / risk
- **Confidence**: high
- **Diff**: small (~30 LOC)
- **Location**: commands/session.rs:48-51 and session/persistence.rs:21-28
- **Today**: (1) **No schema version** — future field rename has no migration. (2) **No atomic write** — mid-process-kill leaves truncated `sessions.json`; next launch zero sessions, all worktrees orphaned. (3) **No validation** — frontend bug producing `"[]"` silently destroys metadata.
- **Proposed**: Wrap as `{"version":1,"sessions":[...]}`; do not delete on unknown. Atomic write: `.tmp` → fsync → rename. Reject empty / non-round-trip JSON.

## Cross-cutting concerns

1. **SessionManager reach (F13)**: every command in commands/session.rs and RunEvent::Exit handler depend on lock topology.
2. **sysinfo adoption (F1, F2, F3)**: collapses Windows Toolhelp32 + Linux /proc + wmic + pgrep into one cross-platform path. metrics.rs already uses sysinfo.
3. **"NO caching" rule violation in path_resolve.rs (F10)**: CLAUDE.md is unambiguous. Either delete cache or get rule amendment.
4. **YAML-adjacent SKILL.md parser (F19)**: shared by Claude and Codex skill loaders.
5. **wmic deprecation (F2)**: Windows 11 24H2 default install lacks wmic.exe.
6. **Schema versioning (F14)**: pattern of `{ version, payload }` should apply wherever we serialize state.
