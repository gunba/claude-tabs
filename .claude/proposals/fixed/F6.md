## Findings

### F6-1 — Concrete dead exports (delete with user confirmation per CLAUDE.md rule 3)
- **Category**: dead-code
- **Confidence**: high
- **Diff**: small (~30 LOC removed)
- **Today**: Searched entire `src/` tree. The following are unused outside their definition site or tests:
  - `extractModelFamily` (lib/claude.ts:157) — thin wrapper, no callers
  - `isModel1m` (lib/claude.ts:162) — one-liner, no callers
  - `forceSessionColor` (lib/claude.ts:331), `getSessionColorIndex` (:326) — leftover restoration helpers
  - `dumpTraces`, `getTraces`, `__perfDump`, `__perfEntries` (lib/perfTrace.ts:188-209) — `traces` ring buffer never read; tests don't import
- **Proposed**: Delete after user confirmation.

### F6-2 — Inspector hook stale-closure dance is fragile
- **Category**: simplify
- **Confidence**: medium
- **Diff**: small (~15 LOC change)
- **Location**: useInspectorConnection.ts:148
- **Today**: `connectRef = useRef(() => {})` initialized with no-op; `connectRef.current = connect` runs every render — silent contract that assignment must occur before next setTimeout fires. `sessionIdRef` and `connectRef` are workarounds for stale closures.
- **Proposed**: Add `useStableCallback` helper (~10 LOC). Rewrite `connect` as stable callback. Drops `connectRef` and `sessionIdRef`. Keep `everConnectedRef` (correct ref, captures real state).

### F6-3 — Retry constants should move to uiConfig.ts
- **Category**: maintainability
- **Confidence**: medium
- **Diff**: small
- **Today**: `INITIAL_RETRY_MS=100`, `MAX_INITIAL_RETRIES=30`, `RECONNECT_DELAYS=[2000, 4000, 8000]` hardcoded. Not "arbitrary delays as control flow" — they're exponential backoff with bounded retry. Concern is observability.
- **Proposed**: Hoist to `lib/uiConfig.ts` under `inspector: { initialRetryMs, maxInitialRetries, reconnectDelaysMs }`. Pure config move.

### F6-4 — Discovery hooks: composition is verbose
- **Category**: simplify
- **Confidence**: high
- **Diff**: small (~40 LOC saved)
- **Location**: useCommandDiscovery.ts (3 chains), useCliWatcher.ts (2 checkClaude/checkCodex)
- **Today**: Three deeply-nested IIFE-style chains do `traceAsync(...).then(log).catch(log)` for each invoke. Each chain ~25 lines. Three of them.
- **Proposed**: Helper `discoverCommands(invokeName, args, perfEvent, loadedEvent, failedEvent)` collapses to ~15 lines. Same for useCliWatcher.

### F6-5 — `tomlHighlight` smol-toml AST not viable — KEEP AS-IS
- **Category**: clarification
- **Confidence**: high
- **Today**: smol-toml exposes `parse()` only — no token/AST stream suitable for syntax highlighting. Switching would require separate pass for tokens. No suitable alternative dep in package.json.
- **Proposed**: No change. The current line-by-line tokenizer is correct, well-commented, 224 LOC.

### F6-6 — perfTrace vs perfTelemetry boundary
- **Category**: simplify
- **Confidence**: medium
- **Diff**: small
- **Today**: `perfTrace.ts` exports `trace`/`traceAsync`/`traceSync`/`startTraceSpan` — call-site instrumentation. `perfTelemetry.ts` consumes `getTraceStats()` from perfTrace.
- **Proposed**: Drop `traces[]` ring buffer (Section F6-1 #4) AND `getTraceStats` consumer in perfTelemetry. Perf data flows through dlog.

### F6-7 — `claude.ts` bloat and module split
- **Category**: refactor
- **Confidence**: medium
- **Diff**: medium
- **Today**: 375 LOC mixing 5 concerns: resume/respawn, model registry, color theming, heat-level computation, token formatting.
- **Proposed**: Split into `claudeResume.ts`, `modelRegistry.ts`, `tabColors.ts`, `heat.ts`. Add `claude.ts` re-exports for one release.

### F6-8 — `usePty.ts` 84 LOC adds little; `ptyRef` and `handle` return are dead
- **Category**: dead-code / inline
- **Confidence**: medium
- **Diff**: small
- **Today**: `ptyRef` (line 19) is set but never read; `handleRef.current` is returned but `TerminalPanel` doesn't use it.
- **Proposed**: Inline `usePty` into TerminalPanel. Shrinks TerminalPanel by ~15 LOC, drops one indirection. Defer to F7's TerminalPanel review.

### F6-9 — `useNotifications.ts`: duplicated cooldown / flash code
- **Category**: simplify
- **Confidence**: high
- **Diff**: small
- **Location**: settled-state path L78-115, error path L121-143
- **Today**: Two near-identical blocks for permission check + session find + cooldown + invoke + flash.
- **Proposed**: Extract `notifySession(sessionId, title, body, kind)` helper. Saves ~20 LOC, single-source for cooldown rule. Switch `lastNotifyRef` from `Record` to `Map` and prune entries when sessions removed.

### F6-10 — `useCliWatcher.ts` line 84: transient help failure wipes capabilities
- **Category**: correctness (likely)
- **Confidence**: medium
- **Diff**: trivial
- **Today**: When `get_cli_help` fails but `check_cli_version` succeeds, calls `setCliCapabilitiesForCli("claude", version, emptyCapabilities())` — silently strips a previously-good capabilities set on a transient failure.
- **Proposed**: Preserve cached capabilities, only update version. **Ask user** if this is intentional.

### F6-11 — `inspectorHooks.ts` (1025 LOC): tap-injection script
- **Category**: tech-debt
- **Confidence**: medium
- **Diff**: medium
- **Today**: Single giant template string of JS evaluated in Bun runtime. No syntax checking. 22 tap categories but `tapToggleAllExpr` (L1024) inlines all 20 toggleable category names literally — three places must be updated when a new category is added.
- **Proposed**: Extract to `inspectorHooks/installTaps.js` (raw source), import via `?raw` (Vite supports). Run minimal `acorn` parse in vitest test. ~5 LOC test, prevents typo regressions. Static const `TOGGLEABLE_CATEGORIES = [...]` driven from same source as type.

### F6-12 — Minor wins
- `formatMetrics.ts` (26 LOC): comment claims "green/<30%" but `cpuColor()` returns `--text-secondary`. Doc-only mismatch — pick one.
- `uiConfig.ts:95`: `deepMerge` duplicated from settings store. Consolidate if shared utility exists elsewhere.
