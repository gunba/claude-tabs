# F2: settings + runtime stores

## Files in scope
- `src/store/settings.ts` (1193 LOC)
- `src/store/runtime.ts` (49 LOC)

Settings is the largest store (~70 fields, ~40 actions, 288 consumer references). Runtime is small, focused, and not in need of structural change.

## Findings

### F2-1: Legacy single-CLI mirrors are dead weight — collapse them in v24
- **Category**: dead code / state shape
- **Confidence**: HIGH
- **Diff size**: M
- **Location**: `src/store/settings.ts:210-226, 319-333, 497-504, 533-536, 686, 743, 787, 1031-1054, 1099-1117, 1147-1155`
- **Today**: `cliVersion`, `previousCliVersion`, `cliCapabilities`, `slashCommands`, `binarySettingsSchema`, `settingsJsonSchema`, `knownEnvVars` are all kept in state and persisted as legacy mirrors of their `*ByCli` siblings. Live consumer audit:
  - `knownEnvVars` (legacy): 0 non-test consumers.
  - `recordingConfig` (legacy): 0 non-test consumers.
  - `slashCommands` (legacy): 0 non-test consumers.
  - `cliVersion`, `previousCliVersion`, `cliCapabilities`: only one use at `StatusBar.tsx:441`.
  - `binarySettingsSchema`, `settingsJsonSchema`: read by `SettingsPane.tsx` and `SettingsTab.tsx`, both already destructure `settingsSchemaByCli` alongside.
- **Proposed**: (1) v24 migration drops legacy mirror keys. (2) Delete fields from `SettingsState`. (3) Migrate the 4 consumer call-sites. (4) Delete `setCliCapabilities`/`setSlashCommands` shims. Win: ~80 fewer LOC, ~20% smaller persisted blob.
- **Side effects**: One-version partialize cleanup.

### F2-2: Bulk-destructured `useSettingsStore()` calls subscribe components to every state change
- **Category**: performance / re-render hygiene
- **Confidence**: HIGH
- **Diff size**: S
- **Location**: `SessionLauncher.tsx:84-85`, `SettingsPane.tsx:242`, `SettingsTab.tsx:71`, `EnvVarsTab.tsx:58`, `ResumePicker.tsx:105`
- **Today**: Five components call `useSettingsStore()` with no selector. Hot paths re-render: `commandRefreshTrigger`, `commandUsage`, `pastSessionsLoading` flips, `recordingConfigsByCli`, `observedPrompts`.
- **Proposed**: Replace with explicit selectors. Pattern already used elsewhere (e.g. `App.tsx`).
- **Side effects**: None on behaviour. Pure render-count reduction.

### F2-3: `quietRecordingConfig` is a one-time normalizer that has already run
- **Category**: code clarity / dead branch
- **Confidence**: MEDIUM-HIGH
- **Diff size**: XS
- **Location**: `:76-104, 1091-1098`
- **Today**: Referenced only from the `version < 21` migration; not used by any `set` paths.
- **Proposed**: Inline body into the v21 branch and delete the helper plus `HIGH_VOLUME_TAP_CATEGORIES` array (only used here).

### F2-4: `cloneRecordingConfig`'s default-arg conflates "build a fresh default" with "deep-clone existing"
- **Category**: design / API hygiene
- **Confidence**: MEDIUM
- **Diff size**: S
- **Location**: `:82-93`
- **Today**: Two unrelated jobs in one function. Silent backfill at L86 means call-sites that "just clone" mutate the schema.
- **Proposed**: Split into `defaultRecordingConfig(cli: CliKind = 'claude')` (parameterized builder) and a pure `cloneRecordingConfig(config)` (or `structuredClone`). Migrations explicitly handle new-category backfill.

### F2-5: Per-CLI setters duplicate the legacy mirror dance
- **Category**: design / repetition (subsumed if F2-1 lands)
- **Confidence**: MEDIUM
- **Diff size**: XS
- **Today**: Six setters all hand-roll "write `byCli[cli]`, conditionally mirror to legacy if `cli === 'claude'`". `setRecordingConfig` always mirrors to `recordingConfigsByCli.claude`, ignoring any param.
- **Proposed**: Best — drop legacy mirrors (F2-1).

### F2-6: Slice the store — extract three coherent sub-stores
- **Category**: architecture / aggressive design
- **Confidence**: MEDIUM
- **Diff size**: L
- **Today**: Single 1193-LOC store mixes four orthogonal concerns: discovery cache, recording/observability, persisted UI prefs, transient UI state.
- **Proposed**: Three Zustand stores: `useDiscoveryStore`, `useRecordingStore`, `useSettingsStore` (shrinks to ~400 LOC). One-shot import migration copies legacy `code-tabs-settings` payload into the three new keys.
- **Side effects**: 288 consumer call-sites need import-target shifts.

### F2-7: Debug-capture subscription bypasses `subscribeWithSelector` and mis-reads the resolver intent
- **Category**: correctness + performance
- **Confidence**: HIGH
- **Diff size**: XS
- **Location**: `:1184-1193`
- **Today**: Resolver `resolveDebugCaptureForSession` (L1175) considers Claude OR Codex `debugCapture`. Top-level subscriber only watches Claude — if user toggles Codex `debugCapture` on while Claude is off, global flag stays `false`. Real correctness gap. Subscriber attaches to entire store and runs on every `set()` from any of the ~40 actions.
- **Proposed**: Wrap with `subscribeWithSelector` middleware: `subscribe((s) => s.recordingConfigsByCli.claude.debugCapture || s.recordingConfigsByCli.codex.debugCapture, setDebugCaptureEnabled)`.

### F2-8: `setSavedDefaults` and `cacheSessionConfig` disagree on whitelist semantics
- **Category**: maintainability / latent bug surface
- **Confidence**: MEDIUM
- **Diff size**: XS
- **Location**: `:425-450, 581-610`
- **Today**: 17-field whitelist copied positionally. The whitelist drifts: `cacheSessionConfig` whitelists `cli` only when `=== 'codex'` (L584), opposite of `setSavedDefaults` (always sets `cli`).
- **Proposed**: Extract `pickWorkspaceFields(config: SessionConfig): Partial<SessionConfig>` constant in `src/types/session.ts`.

### F2-9: Migrations interleave version-blind defensive shims
- **Category**: code archaeology
- **Confidence**: MEDIUM
- **Diff size**: S
- **Location**: `:945-1131`
- **Today**: 23 linear migrations. v9 and v14 are no-op placeholders. v19/v20/v21/v22 each test "if `byCli` not present, build from legacy".
- **Proposed**: Compact v9/v14 to a single comment block. Extract `rebuildRecordingConfigsByCli(state)` and `rebuildPerCliCaches(state)` helpers. Add unit test asserting v0→latest migration round-trips against a golden fixture.

### F2-10: `loadBinarySettingsSchema` / `loadSettingsJsonSchema` / `loadKnownEnvVars` are name-only forwarders
- **Category**: dead code (subsumed into F2-1)
- **Confidence**: HIGH
- **Diff size**: XS
- **Location**: `:652-660`
- **Proposed**: Migrate `useCliWatcher` to call the per-CLI variants directly and delete the wrappers.

### F2-11: `runtime.ts` minor — `loadRuntimeInfo` does double duty
- **Category**: separation of concerns
- **Confidence**: LOW-MEDIUM
- **Diff size**: XS
- **Location**: `runtime.ts:28-44`
- **Today**: `loadRuntimeInfo` stashes `observabilityInfo` AND starts/stops `startObservabilityBridge` + `startFrontendPerfTelemetry`.
- **Proposed**: Move telemetry start/stop into a `useEffect` in `App.tsx` that observes `observabilityInfo.observabilityEnabled`.

### F2-12: `observedPrompts` is in state but missing from `partialize`
- **Category**: persisted-state correctness
- **Confidence**: MEDIUM
- **Diff size**: trivial
- **Location**: `:344` (state), `:1133-1166` (partialize, absent)
- **Today**: `observedPrompts` is mutated by `addObservedPrompt`, capped at 50, surfaced by `PromptsTab.tsx` as `allObservedPrompts`. Not in `partialize`. The cap-at-50 + `firstSeenAt` timestamps + dedup-by-text + FIFO eviction strongly imply intended persistence.
- **Proposed**: Either add to partialize (matches user-facing intent) or remove the cap/timestamp/dedup logic. Decision belongs with the user.
