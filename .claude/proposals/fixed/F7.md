## Findings

### F7-1 — TerminalPanel: 24-Effect Decomposition
- **Category**: refactor / cohesion
- **Confidence**: high
- **Diff**: large
- **Today**: 657-line function body. 18 effects/callbacks plus 6 mutable refs. `triggerRespawnRef.current = (config, name) => {...}` (L332) reassigned on every render — silent contract.
- **Proposed**: Decompose into four cohesive hooks:
  - `useTerminalSetup(session, terminal, respawnCounter, inspectorPort)` — owns spawn effect (L432-611)
  - `useSessionRespawn({ session, pty, inspector, terminal })` — eliminates `triggerRespawnRef` reassignment-every-render
  - `useSessionDeath({ session, pty, terminal })` — owns `handlePtyExit` (L255-324). Drop unused `session.config.resumeSession`/`sessionId` deps to stable `session.id`
  - `useInspectorSetup(session)` — owns `useInspectorConnection`, `inspectorReconnectKey`, `registerInspectorCallbacks`, connection-derived loading
- **After**: TerminalPanel ~150 LOC. 4 named hooks, each ~3 effects.

### F7-2 — StatusBar: sub-component recompute storm
- **Category**: performance
- **Confidence**: high
- **Diff**: medium
- **Today**: `SessionStatus` (L71-314) rebuilds 14+ tooltip strings on every render. IIFEs at L144, 166, 178, 190, 228, 267 each construct 5-line concatenations using `toLocaleString()`/`Math.round()`. `parseWorktreePath(session.config.workingDir)` at L87 parses on every render. `aliveCountRef.current = aliveSessions.length` at L391-392 is a side effect during render — unsafe under React 19 concurrent. The ref is never read in the file (dead state).
- **Proposed**:
  - Wrap `SessionStatus` in `React.memo` with comparator `sessionStatusPropsEqual(prev, next)` listing each read field (mirrors `terminalPanelPropsEqual` pattern at L31).
  - Hoist sub-IIFEs to dedicated memo-wrapped sub-components: `<TokenChip>`, `<BudgetChip>`, `<CpuMemChip>`, `<ContextDebugChip>`.
  - Delete `aliveCountRef` (dead).
  - Single `useMemo([sessions, subagentMap])` returning `{ aliveSessions, activeSessions, allSessionsTokens }` so the three quantities derive together.
  - Subscribe to store slices, not whole `s.sessions`. Use `useShallow` or split into `(s) => s.sessions.length`, `(s) => s.activeTabId`, separate hook for active session.

### F7-3 — ActivityPanel: 9 useMemos cascade per `activity` reference change
- **Category**: performance / architecture
- **Confidence**: high
- **Diff**: medium-large
- **Today**: 9 useMemos (`lastMainAgentFile`, `lastSubagentFiles`, `activeSubagentIds`, `activeAgentFiles`, `primaryActive`, `contextFileMap`, `fileMap`, `tree`, `rows`) collectively re-derive on `activity` reference changes. **Each `addFileActivity` re-walks every turn × every file × every subagent.** Auto-expand effect at L431-435 calls `mergeExpandedPaths` each time `tree` changes referentially — possible loop. Mascot positioning effect at L450-487 measures DOM via `getBoundingClientRect()` after every render where deps change — JS layout phase.
- **Proposed**:
  - **Split `activity` selector into narrow slices**: separate selectors for `turns`, `allFiles`, `visitedPaths`, `contextFiles`, `lastUserMessageAt`, `expandedPaths`. `lastMainAgentFile`'s `[activity]` dep becomes `[turns, allFiles]`.
  - **Move `lastMainAgentFile` and `lastSubagentFiles` derivation into the activity store** as `derived` selector computed in reducer when files added. O(1) per add vs current O(n) per render. **Not caching** — moving derivation to where data changes.
  - **Wrap `FileTreeRow` in `React.memo`** with explicit comparator on `node`, `depth`, `isExpanded`, agents (shallow), `contextInfo`.
  - **Stabilize `onToggle`/`onFileClick`**: capture `activeTabId` via ref instead of dep.
  - **Replace mascot positioning effect** with `IntersectionObserver`/`ResizeObserver` pair watching for `data-path` changes, eliminating per-file-activity layout thrash.

### F7-4 — HeaderActivityViz: actually in good shape; minor wins
- **Category**: clarification + minor perf
- **Confidence**: medium
- **Diff**: small
- **Today**: Briefing was wrong — `getComputedStyle` is **not** called every render. L251-260 sets up MutationObserver on `document.documentElement` style attr. ResizeObserver in use at L277-280. `renderFrame` constructed once in L262 effect with `[]` deps, reads from refs. `slotsMapRef` cleanup runs at L222 (`for (const id of [...map.keys()]) if (!seen.has(id)) map.delete(id);`).
- **Genuine improvements**:
  - **The 500ms `setInterval` at L246** reads `useSessionStore.getState().sessions` and computes deltas — this is polling timer. Per "no arbitrary delays" rule, code smell. Replace with `useSessionStore.subscribe` callback that runs only when `sessions` changes.
  - **`lastThemeBump` mechanism** (L275, L310-313): cleaner if theme observer calls a stable `markThemeChanged()` flipping `needsResize` directly, eliminating `themeBumpRef` integer counter.
  - **`slots` useMemo** (L163-188) rebuilds on `[sessions, subagents]` reference change. Replace with direct synchronous derivation inside L196 effect.
  - **Render loop** (L309-457): does string concat per frame, `Float32Array` allocation per frame. Allocate once and reuse via `cols`-keyed reset. Minor GC pressure win.

### F7-5 — Header & RightPanel: minor
- **Header.tsx** (118 LOC) is clean — drag threshold per [PL-03] correct. No changes.
- **RightPanel.tsx** (71 LOC): one effect at L27-31 does side-effect `setRightPanelTab("response")` from inside render-derived state — correct as-is for `debugBuild` reconciliation.
- **Proposed**: Wrap `RightPanel` in `React.memo` (no comparator needed — props empty) so far-away store changes don't re-render right column.

## Estimated impact

- TerminalPanel: 24 effects → 4 hooks of 3 each. Maintainability win, not perf.
- StatusBar: per-TAP-event re-render cost from full subtree to delta-only chips. Conservatively 50-70% fewer renders.
- ActivityPanel: `addFileActivity` cost from O(turns × files) per render to O(1) reducer-time + O(visible rows) memo'd render.
- HeaderActivityViz: eliminates 500ms polling timer; reduces per-frame allocations.
