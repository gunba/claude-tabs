# R8: pty/*

## Files in scope (1018 LOC)
- src-tauri/src/pty/mod.rs (344) — Tauri command surface, `Session` struct, cross-platform shim
- src-tauri/src/pty/conpty.rs (380, Windows) — ConPTY pipe pair, attribute list, RAII handles, reader thread
- src-tauri/src/pty/unix.rs (294, Unix) — openpty + Command::pre_exec, RAII fd pair, reader thread, PGID kill

## Findings

### F1 — Heterogeneous lock primitives on a single conceptual session
- **Category**: design / concurrency
- **Confidence**: HIGH
- **Diff**: large (~250 LOC moved, behaviour preserved)
- **Location**: pty/mod.rs:35-50
- **Today**: `Session` mixes four primitives that conceptually move together — `tauri::async_runtime::Mutex<Box<dyn Write+Send>>` (writer), `tauri::async_runtime::Mutex<mpsc::Receiver<Vec<u8>>>` (output_rx), `std::sync::Mutex<UnixPty>` (Unix only), `AtomicU16`x2 (cols/rows), and a duplicated `RawFd` (master_fd) accessed lock-free from `pty_resize`. `ConPtyHandle` internal handles accessed without external serialisation. Three locks async-aware, one blocking. Blocking one is acquired in `pty_kill` (L250) on async runtime worker — blocks executor for `kill(2)` duration.
- **Proposed**: Actor task that owns mutable state (`SessionInner { backend, output_rx, ... }`) with typed message enum (`Write`, `Resize`, `Kill`, `Exit`, `Drain`) and oneshot ack channels. Tauri commands shrink to one send + one await. Multiplex via `tokio::select!`. `wait()` delegates to `spawn_blocking` with completion via oneshot.
- **Side effects**: All seven Tauri commands shrink to ~5 LOC. `pty_exitstatus` no longer blocks any other operation. Enables F2 + F4. Cleanup is by dropping `cmd_tx`.

### F2 — No `PtyBackend` trait — `cfg`-gated branches scattered through `mod.rs`
- **Category**: design / portability
- **Confidence**: HIGH
- **Diff**: medium
- **Location**: 17 `#[cfg(windows)]` / `#[cfg(unix)]` blocks
- **Today**: Every command branches on `#[cfg]`. Both backends share identical shape (`spawn`, `resize`, `kill`, `wait`, `process_id`) but no formal interface exists.
- **Proposed**: `trait PtyBackend: Send + Sync + 'static { fn resize/kill/wait/process_id ... }` returning typed `PtyError`. `mod.rs` uses `#[cfg(windows)] use conpty as platform; #[cfg(unix)] use unix as platform;` and calls `platform::spawn(...)` once.

### F3 — `pty_read` lacks backpressure to frontend; only `pty_drain_output` uses a timer
- **Category**: performance / no-arbitrary-delays rule
- **Confidence**: MED
- **Diff**: small
- **Location**: pty/mod.rs:159-187 and 319-344
- **Today**: Bounded `sync_channel(64) × 8 KiB chunks` = at most 512 KiB before reader thread blocks on `send` (good — propagates pressure to OS pipe). But `pty_read` has no per-call deadline, and `pty_drain_output` (L319-344) hard-codes 500 ms shutdown deadline with 10 ms `recv_timeout` — only place a timer leaks into this module. Per global no-arbitrary-delays rule should be replaced by message-driven shutdown.
- **Proposed**: Replace `mpsc::sync_channel` with `tokio::sync::mpsc::channel(64)`; reader thread uses `tx.blocking_send`. Async command does `recv().await` directly — removes `Mutex<Receiver>` and `spawn_blocking` from `pty_read`. For drain on shutdown: drop reader's `tx` to close channel, then `while rx.recv().await.is_some() {}`. No timer; reader exits when OS pipe closes. Keep 256 KiB per-batch cap (PT-27) using `recv_many`.

### F4 — String errors everywhere (`Result<_, String>`)
- **Category**: design / observability
- **Confidence**: HIGH
- **Diff**: small
- **Today**: Errors are formatted strings; frontend cannot discriminate "PID not found" from HRESULT failure except by string matching. `"Unavailable pid"` duplicated in seven commands.
- **Proposed**: `#[derive(thiserror::Error, serde::Serialize)] #[serde(tag="kind")] enum PtyError { SessionNotFound(PtyHandler), Write(String), Resize(String), Spawn(String), Wait(String), Eof, JoinFailed(String) }`. Frontend matches on `error.kind`.

### F5 — `pty_kill` blocks on async runtime worker (Unix)
- **Category**: concurrency / runtime hygiene
- **Confidence**: HIGH
- **Diff**: trivial
- **Location**: pty/mod.rs:249-250
- **Today**: `session.pty.lock().unwrap().kill()?` is blocking mutex acquisition + syscall on tauri async worker. Counter-evidence: `UnixPty::wait` (unix.rs:104-108) drops the slot lock before `child.wait()` — so the **outer** `Mutex<UnixPty>` is released. Still hazardous; inner `Mutex<Option<Child>>` does remain held across `child.wait()`.
- **Proposed**: Under F1's actor, `pty_kill` becomes `cmd_tx.send(Kill).await`. Smaller patch absent F1: wrap kill in `spawn_blocking` and refactor `UnixPty::kill` to take `&self` reading pid+pgid from atomics.

### F6 — Reader thread is detached and unjoinable
- **Category**: lifecycle
- **Confidence**: MED
- **Diff**: small
- **Location**: conpty.rs:358-372, unix.rs:265-279
- **Today**: `std::thread::spawn(move || ...)` — handle dropped. Thread exits when OS pipe yields EOF or when `output_tx.send` returns `Err`. No `JoinHandle` → cannot detect stuck reader, errors swallowed (`Err(_) => break`).
- **Proposed**: Capture `JoinHandle` in `SpawnResult`; store on `Session`; join during actor shutdown. Change channel type to `mpsc::Sender<Result<Vec<u8>, ReaderError>>` so stuck-then-broken pipe surfaces as error rather than silent EOF.

### F7 — `Session::cols`/`rows` atomics written but never read
- **Category**: dead code
- **Confidence**: HIGH
- **Diff**: trivial
- **Location**: pty/mod.rs:48-49 (decl), 105-106 (init), 204-205 (resize). No reads anywhere.
- **Today**: Frontend tracks size in TerminalPanel.tsx:391 and xterm.js dimensions; backend has no need.
- **Proposed**: Per "ask before deleting" — recommend deletion.

### F8 — Windows env block omits TERM/TERM_PROGRAM/COLORTERM defaults
- **Category**: portability / determinism
- **Confidence**: HIGH
- **Diff**: small
- **Location**: conpty.rs:207-229 vs unix.rs:209-219
- **Today**: Windows `build_env_block` inherits parent env + overlays caller env but does **not** inject `TERM=xterm-ghostty`, `TERM_PROGRAM=ghostty`, `COLORTERM=truecolor`. Unix injects all three before caller env. Claude Code's `isSynchronizedOutputSupported()` env-sniff (PT-23) will not see `xterm-ghostty` on Windows.
- **Proposed**: Move TERM/TERM_PROGRAM/COLORTERM injection into `pty/mod.rs::pty_spawn` so both backends apply identical defaults.

### F9 — `pty_read` / `pty_destroy` race can drop queued data
- **Category**: concurrency / correctness
- **Confidence**: MED
- **Diff**: trivial after F1
- **Location**: pty/mod.rs:159-187
- **Today**: `pty_read` holds `output_rx.blocking_lock()`. If `pty_destroy` lands between two reads, second fails with `"Unavailable pid"` while data may still be queued.
- **Proposed**: Under F1+F3, actor is single consumer of receiver. `pty_read` sends `Read { ack }` message; cancellation = drop oneshot. On destroy, drain channel inside actor before exit.

### F10 — Reader thread `Err(_) => break` swallows IO errors
- **Category**: observability
- **Confidence**: HIGH
- **Diff**: trivial
- **Location**: conpty.rs:362-371, unix.rs:268-278
- **Today**: Any `read` error other than EOF terminates thread silently. `pty_read` then sees `mpsc::RecvError` and returns `Err("EOF")`. Frontend cannot distinguish broken pipe from permission denied.
- **Proposed**: Forward `io::Error` through channel as typed `ReaderError` (F6); surface as typed `PtyError` (F4); log via `record_backend_event` with WARN.

### F11 — `unix.rs` is not macOS-clean
- **Category**: portability
- **Confidence**: MED
- **Diff**: small
- **Today**: File named `unix.rs`, gate is `#[cfg(unix)]`. `PR_SET_PDEATHSIG` is correctly Linux-gated (L238). But rest of spawn path (`setsid`/`TIOCSCTTY`, `pre_exec` semantics) has macOS pitfalls. User CLAUDE.md mandates Win+Linux; macOS unspecified.
- **Proposed**: Ask user. If macOS is target, add CI matrix entry. If not, gate `unix.rs` to `#[cfg(target_os = "linux")]` and `compile_error!` for other Unix variants.

### F12 — ConPTY reader has no read deadline
- **Category**: liveness
- **Confidence**: MED
- **Diff**: small
- **Location**: conpty.rs:60-74, 362
- **Today**: ConPTY `ReadFile` blocks indefinitely. On Ctrl+C, child exits but ConPTY may not close pipe. Backend reader thread can be stuck in `ReadFile` forever. Rely on `ClosePseudoConsole` from `OwnedHpcon::drop` (L86) to unblock — happens on `pty_destroy`. 500ms timer is only thing bounding destroy path.
- **Proposed**: Tied to F3 — drop the timer; let drain run until channel closes.

### F13 — `_session_id: Option<String>` unused
- **Category**: dead code
- **Confidence**: HIGH
- **Diff**: trivial
- **Location**: pty/mod.rs:57
- **Today**: Backend accepts `_session_id` but never uses it; frontend always passes it. Backend `record_backend_event` calls do not include session id.
- **Proposed**: Ask user. Either wire it into `record_backend_event(..., session_id)` so backend events are scoped per-session, or drop it.

### F14 — No Rust unit tests for pty/
- **Category**: testing
- **Confidence**: HIGH
- **Diff**: medium
- **Today**: PTY tests are TypeScript (`ptySpawn.test.ts`, `ptyCleanup.test.ts`, `ptyProcess.test.ts`) and mock IPC at boundary. Rust pre-commit gate runs `cargo test` but exercises nothing in this module. F1, F5, F9 hazards cannot be regression-tested.
- **Proposed**: With F2's `PtyBackend` trait, add `MockPtyBackend` driven by `tokio::sync::mpsc`. Cover: actor responds to Resize while Wait in flight (F1+F5); destroy-during-read returns typed `SessionNotFound` (F4+F9); reader-thread error surfaces as typed `ReaderError` (F6+F10); drain completes deterministically without timer (F3).

## Cross-cutting concerns

### usePty hook (F6 frontend scope)
F4's typed `PtyError` ripples into TS via Tauri's discriminated-union error serde, letting `usePty` distinguish "session vanished" from "kill failed".

### TerminalPanel (F7 frontend scope)
After F1's actor refactor, resize is single message send; `master_fd` aliasing trick disappears, cols/rows atomics in F7 disappear.

### `ptyProcess.ts` parallel exit waiter (PT-10)
Frontend issues two concurrent `invoke('pty_exitstatus')` per session. Backend currently supports via `child.wait()` `take()` semantics; second caller gets `"wait already consumed"`. Under F1, multiple waiters share a `tokio::sync::watch::channel::<Option<u32>>`.

## Out of scope

- Frontend write-batching pipeline (`terminalWriteQueue`, `useTerminal.writeBytes`).
- Process-tree teardown registry (R10's `commands/process.rs`).
- Spawn-time discovery (PT-19/PT-23 invariants).
- `record_backend_event` observability layer.
