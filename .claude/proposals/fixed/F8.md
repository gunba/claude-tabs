Cross-checked: SessionConfig (22 fields) is held in **one** `useState<SessionConfig>` plus 8 ancillary `useState`s — the assignment's "28 useStates" reads each SessionConfig field as a separate slot. Actual count is `setConfig` plus 9 small flags.

## Findings

### F8-1 — Unselected Zustand subscription forces re-render on every settings store mutation
- **Category**: perf-bug (real)
- **Confidence**: high
- **Diff**: small (~6 lines per file)
- **Location**: SessionLauncher.tsx:84-85, ResumePicker.tsx:105
- **Today**:
  - L84: `const { recentDirs, lastConfig, savedDefaults, workspaceDefaults, setShowLauncher, addRecentDir, removeRecentDir, setLastConfig, setSavedDefaults } = useSettingsStore();`
  - L105 (picker): `const { addRecentDir, setShowLauncher, setLastConfig } = useSettingsStore();`
  - `useSettingsStore()` with no selector subscribes to entire 49KB store. Any mutation re-renders the whole subtree.
- **Proposed**: Replace with explicit per-field selectors. Pattern already used elsewhere (App.tsx).
- **Side-effects**: Zero. Same data, narrower subscription.

### F8-2 — Inline option arrays force Dropdown rerender work
- **Category**: perf
- **Confidence**: high
- **Diff**: small
- **Location**: SessionLauncher.tsx:626, 635, 650, 688, 698
- **Today**: Five Dropdowns pass `options={[{ value: "", label: "default" }, ...modelOptions.map(...)]}` — fresh array per render. `Dropdown.tsx` has **no `React.memo`**.
- **Proposed**: (a) Move arrays into `useMemo`; (b) wrap `Dropdown` and `PillGroup` in `React.memo`.

### F8-3 — `useSessionConfig()` reducer to collapse mega-state
- **Category**: refactor / clarity
- **Confidence**: high
- **Diff**: medium (~150 line new hook, ~80 line component shrink)
- **Location**: SessionLauncher.tsx:93-113, 215-250, 298-308, 398-442
- **Today**: `setConfig` invoked from 12+ sites. `applyWorkspaceDefaults` (L216) duplicates `buildInitialLauncherConfig` logic. Two separate `useEffect`s (L298, L304) re-validate model/effort against adapter lists.
- **Proposed**: `useReducer` with named actions: `set`, `switchWorkspace`, `switchCli`, `applyResumeBase`, `clearOneShot`, `validateAdapterOption`. Reuse `buildInitialLauncherConfig`.

### F8-4 — Content-search has no Rust-side cancellation
- **Category**: perf-bug
- **Confidence**: high (verified `src-tauri/src/commands/data.rs:900-909`)
- **Diff**: medium (Rust + TS)
- **Location**: ResumePicker.tsx:121-164; data.rs:900
- **Today**: Frontend `searchCounterRef` discards stale results, but Rust `search_session_content_sync` walks `~/.claude/projects/` + `$CODEX_HOME/sessions` to completion (≤20 MB per file, ≤50 results). No `AbortSignal`, no shared cancel flag. Unmount cleanup at L161 only clears debounce timer — the in-flight `invoke()` keeps blocking the Tauri runtime.
- **Proposed**:
  1. `useContentSearch(query)` hook with `AbortController`-style controller-ref.
  2. Add `cancel_token: String` (UUID) param to Rust command. Maintain `Mutex<HashMap<String, Arc<AtomicBool>>>` in command module; new tokens flip prior tokens to true; scan loop checks flag at file boundaries.

### F8-5 — `usePastSessionChains()` extraction; "race risk" claim is misdiagnosed
- **Category**: refactor + clarification
- **Confidence**: high
- **Diff**: medium
- **Location**: ResumePicker.tsx:217-301, 304-310, 313-351
- **Today**: Three large `useMemo`s. **Correction**: `pastSessions` is read via `useSettingsStore((s) => s.pastSessions)` at L107 — **not a prop**. There is no race. Real issue is the test file (`resumePicker.test.ts` L29-75, L297-410) duplicates the merge/filter algorithms as second-copy pure functions.
- **Proposed**: Extract `usePastSessionChains({ pastSessions, dirFilter, sessionNames, deadSessionMap, contentResults })` returning `{ filteredPastSessions, mergedList, displayList, contentDividerIndex, snippetMap }`. Tests `import` from hook module.

### F8-6 — `useDirectoryValidation()` + launch in-flight guard
- **Category**: refactor + small bug surface
- **Confidence**: medium-high
- **Diff**: small
- **Location**: SessionLauncher.tsx:407-414, 464-471
- **Today**: `handleLaunch` is async with `await invoke("dir_exists", ...)` mid-flow. Keydown listener at L466 fires Enter without in-flight guard — concurrent Enter could double-launch.
- **Proposed**: (a) Extract `useDirectoryValidation()` for blur-time validation feedback; (b) add `launchingRef` set on entry, cleared in `finally`.

### F8-7 — `<CliOptionGrid>` sub-component
- **Category**: refactor
- **Confidence**: medium
- **Diff**: small
- **Location**: SessionLauncher.tsx:746-826
- **Today**: 80 lines of pill-grid JSX. Click handler closures depend on `commandLine`, `config`, `buildFullCommand` — every textarea keystroke re-renders ~30+ pills.
- **Proposed**: Extract `<CliOptionGrid options activeFlags onPillClick heatClasses>` and wrap in `React.memo`. Memoize `heatClasses`.

### F8-8 — Command-line round-trip rebuild + "where did my model go" bug
- **Category**: clarity / latent bug
- **Confidence**: medium
- **Diff**: small-medium
- **Location**: SessionLauncher.tsx:187-191, 372-397, 380-383
- **Today**: Two state slots — `config` (structured) and `commandLine` (string). Effect at L187 rebuilds string from config; `launchConfig` at L372 reverse-parses string. **Crap flag** at L380-383:
  ```ts
  // User edited the generated part too — pass entire command line as extra
  const parts = commandLine.replace(/^(claude|codex)\s*/, "").trim();
  extra = parts || null;
  ```
  When user edits any prefix, structured `config.model` still flows through `createSession` separately. Possible duplicated flags or silent loss. **Worth raising with user before refactoring.**
- **Proposed**: (a) Collapse to single source-of-truth — `extraFlags` lives in `config`, `commandLine` is derived; manual edits flip `manualOverride: boolean`. Or (b) Add invariant test pinning `buildFullCommand(launchConfig) === commandLine`.

### F8-9 — Window keydown listener churn
- **Category**: perf (low-impact)
- **Confidence**: high
- **Diff**: small
- **Location**: SessionLauncher.tsx:464-471, ResumePicker.tsx:445-483
- **Today**: Effect deps include `handleLaunch`/`handleResume`/`displayList`/`selectedIndex` — listener removed and re-added on each callback identity change.
- **Proposed**: Ref-based dispatch — single listener installed once, reads from `handlersRef.current`.

### F8-10 — Per-row allocations inside `displayList.map`
