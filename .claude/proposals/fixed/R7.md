## Key correction to assignment framing

The assignment claims "CliAdapter is enum-dispatched (not dyn-trait). Adding a new CliKind variant could miss implementations." This is **inaccurate**:
- `mod.rs:101-121` defines `pub trait CliAdapter` already.
- `mod.rs:124-129` returns `Box<dyn CliAdapter>` from `adapter_for(kind)`.

What is actually enum-dispatched is the **selector** in `build_cli_spawn` (`mod.rs:150 if cli == CliKind::Codex`) — that block holds two pieces of Codex-only logic (`merged_codex_spawn_env` + `append_codex_proxy_config`) outside the adapter. The trait surface only exposes `build_spawn`/`launch_options`/`detect`/`version` — there is no hook for the adapter to attach its own post-build env merger or arg-injection step.

## Findings

### F-R7-1 — `build_cli_spawn` knows Codex internals (spawn-env sidecar + proxy injection)
- **Category**: design / single-responsibility
- **Confidence**: high
- **Diff**: medium
- **Location**: cli_adapter/mod.rs:136-173
- **Today**: After `adapter.build_spawn` returns a `SpawnSpec`, `build_cli_spawn` inspects `cli == CliKind::Codex` and does two extra mutations: (a) merges `merged_codex_spawn_env` into `spec.env_overrides`; (b) appends `-c model_provider=...` proxy args via `append_codex_proxy_config`.
- **Proposed**: Promote both side-effects to trait methods: `fn post_build_env(&self, app: &AppHandle, cwd: &str) -> Vec<(String, Option<String>)>` and `fn post_build_args(&self, ctx: SpawnContext) -> Vec<String>` where `SpawnContext` carries proxy port + auth_mode + session_id. ClaudeAdapter returns empty defaults; CodexAdapter does merge + proxy injection. `build_cli_spawn` becomes clean dispatch (no `if cli == ...`).

### F-R7-2 — Hardcoded Codex proxy provider; no kill-switch / customization
- **Category**: extensibility
- **Confidence**: high
- **Diff**: small
- **Location**: cli_adapter/mod.rs:175-251
- **Today**: Provider name `code-tabs-proxy`, `wire_api=responses`, `requires_openai_auth=true`, hardcoded headers — all baked into source. No way to (a) disable proxy injection, (b) swap provider name, (c) add custom header.
- **Proposed**: Pull config out of source via reserved env-var prefix in spawn-env sidecar: `CODE_TABS_PROXY_DISABLED=1`, `CODE_TABS_PROXY_PROVIDER_ID=...`, `CODE_TABS_PROXY_HEADERS=OpenAI-Beta:OPENAI_BETA,...`. Read via existing `merged_codex_spawn_env` merge — no new config file.

### F-R7-3 — `run_claude_cli` no internal timeout; tempfile lifecycle has no panic safety
- **Category**: reliability
- **Confidence**: high
- **Diff**: small
- **Location**: commands/cli.rs:145-238
- **Today**: `child.wait()` blocks indefinitely. Callers don't wrap with `tokio::time::timeout`. Hung Claude binary wedges spawn_blocking thread forever. Tempfile removed only on happy `wait()` path; on `wait()` error it leaks. `read_to_end` of stderr is unbounded — misbehaved CLI streaming MB of stderr could OOM.
- **Proposed**:
  - Enforce timeouts at Tauri command boundary via `tokio::time::timeout` wrapping `spawn_blocking`, 30s default for plugin commands.
  - Wrap tempfile in RAII guard `TempStdoutFile { path }` with `Drop { fs::remove_file(&path).ok() }`.
  - Cap stderr capture at 64 KiB via `take(64 * 1024).read_to_end`.

### F-R7-4 — `scan_command_usage_from_roots`: claude path no depth/symlink-cycle protection
- **Category**: correctness / robustness
- **Confidence**: high
- **Diff**: medium
- **Location**: commands/cli.rs:679-815
- **Today**: Asymmetric: `scan_codex_command_usage` uses `collect_jsonl_files(dir, depth=6)`; `scan_claude_command_usage` uses inline two-level walk. `collect_jsonl_files` recurses on `path.is_dir()` which follows symlinks. Asymmetric error recovery: claude propagates `read_dir` errors via `?`; codex swallows.
- **Proposed**: Unify on single `walk_jsonl_files` helper using `WalkDir` from `ignore` crate (already a dep). `.follow_links(false).max_depth(N)` — `WalkBuilder` handles symlink-cycle detection natively. Both passes funnel through it.

### F-R7-5 — Hand-rolled semver parsing in `version.rs` is fragile
- **Category**: design / dependency hygiene
- **Confidence**: high
- **Diff**: medium
- **Location**: commands/version.rs:60-156
- **Today**: Real bugs:
  - `parse_version` returns `None` if string doesn't contain `\d+\.\d+\.\d+`, but `compare_versions` returns `Equal` on `None` (L84-89). So `compare_versions("garbage", "2.1.119") == Equal`, incorrectly **drops** "garbage" entries from changelog selection.
  - No test pins SemVer 2.0.0 §11 ordering for `2.0.0-rc` vs `2.0.0-beta`.
  - Six independent regex compilations per call; `Regex::new` panics on malformed.
- **Proposed**: Add `semver = "1"` (~3 KB compiled). Replace `ParsedVersion`/`parse_version`/`compare_versions`/`compare_prerelease`/`is_prerelease` with `semver::Version` + `Version::cmp`. Use `OnceLock<Regex>` for compile-once patterns.

### F-R7-6 — Changelog HTML/atom parser is regex strings with no entity coverage
- **Category**: correctness / brittleness
- **Confidence**: medium
- **Diff**: medium
- **Location**: commands/version.rs:275-365
- **Today**: `decode_xml_entities` covers only literal substitutions. Numeric entities (`&#x2014;`, `&#8212;` for em-dash) and named entities (`&copy;`, `&hellip;`) pass through unchanged. Real GitHub release notes contain these. `scraper = "0.23"` is **already a dep** (Cargo.toml:78).
- **Proposed**: Replace `html_fragment_to_markdown` + entity decoder with `scraper`-based walker. `scraper::node::Element::text()` decodes all named/numeric entities. For atom feeds, switch to `scraper::Selector`.

### F-R7-7 — `ClaudeAdapter::launch_options` returns hardcoded data that contradicts its docstring
- **Category**: dead-code-or-bug
- **Confidence**: high
- **Diff**: small (deletion or rewrite — **ASK USER**)
- **Location**: cli_adapter/claude.rs:57-118
- **Today**: Function comment says launcher reads from TS-side sources "until batch 4 lands the CodexAdapter and the launcher pivots to adapter-driven options." TS side already calls `cli_launch_options` for both CLIs. Hardcoded list is wired up live but data is fabricated:
  - `models: ["best", "opus", "opus[1m]", "sonnet", "sonnet[1m]", "haiku"]` — CLI shortcuts/aliases, not real catalog
  - `effort_levels: ["low", "medium", "high", "xhigh", "max"]` — `max` is the value `CODEX_EFFORT_VALUES` rejects (codex.rs:31)
  - `flag_pills: Vec::new()` — empty
- **Proposed** (per CLAUDE.md §3 "ASK"):
  - (a) Have `ClaudeAdapter::launch_options` call into real Claude discovery (binary scan) for symmetry with Codex.
  - (b) Delete body and stop registering Claude in `cli_launch_options`, falling back to TS-side discovery.

### F-R7-8 — `detect_install_method` + `update_cli`: substring path heuristic; asymmetric with Codex
- **Category**: correctness / Codex parity
- **Confidence**: medium
- **Diff**: medium
- **Location**: commands/version.rs:497-593
- **Today**: Substring matching `homebrew | linuxbrew | volta | node_modules | claude/versions` — false positives possible. **Codex has no `update_cli` equivalent** — user running both sees asymmetry.
- **Proposed**: Promote `detect_install_method` to ordered table with proper component matching. Add `update_codex_cli` Tauri command (npm-installed → `npm update -g @openai/codex`; cargo-installed → `cargo install codex-cli`).

### F-R7-9 — `extra_flags.split_whitespace` breaks shell-quoted values
- **Category**: correctness / edge case
- **Confidence**: high
- **Diff**: tiny
- **Location**: commands/cli.rs:633 and cli_adapter/codex.rs:138
- **Today**: User typing `--config=instructions="be precise"` produces `["--config=instructions=\"be", "precise\""]` — broken.
- **Proposed**: Add `shlex = "1"` (~10 KB pure Rust). Replace `split_whitespace` with `shlex::split(extra)` which handles posix quoting.

### F-R7-10 — `read_codex_auth_mode_sync` re-reads `auth.json` on every spawn
- **Category**: I/O efficiency (no-op)
- **Confidence**: high
- **Today**: File is small; OS page cache absorbs repeats. User's no-cache rule prohibits memoization. Per-spawn cost negligible.
- **Proposed**: **Leave alone.**

### F-R7-11 — `CodexAdapter::launch_options` runs `codex` binary thrice
- **Category**: I/O efficiency (non-caching restructure)
- **Confidence**: high
- **Diff**: small
- **Location**: cli_adapter/codex.rs:165-248
- **Today**: `discover_codex_models_sync()` and `discover_codex_cli_options_sync()` — sequential subprocess spawns. Each re-resolves binary path via `detect_codex_cli_sync` (which itself runs `which codex`). Three subprocesses per launch_options.
- **Proposed**: Two pure call-graph restructures (no caching): (1) Pass resolved binary path into both discovery calls (signature change). (2) Run two `codex` invocations concurrently via `tokio::task::spawn_blocking` x2 + `tokio::join!`. Latency drops from ~3x detect to ~1x detect + max(model, options) parallel.

### F-R7-12 — `extract_codex_slash_command` JSON-parses every line; no fast-path filter
- **Category**: efficiency
- **Confidence**: medium
- **Diff**: tiny
- **Location**: commands/cli.rs:815-829
- **Proposed**: Cheap byte-substring filter: `if !line.contains("\"user_message\"") { continue; }`. Cuts ~95% of allocations.

### F-R7-13 — Path-separator normalization asymmetric across adapters
- **Category**: cross-platform consistency
- **Confidence**: medium
- **Diff**: small
- **Location**: commands/cli.rs:599-605; cli_adapter/codex.rs:74-77
- **Today**: Claude's `--project-dir` re-emits `working_dir.replace('/', "\\")` on Windows. Codex's `--cd` does not.
- **Proposed**: Move path normalization into shared `crate::path_utils::canonical_for_cli(&str) -> String`. Either both rewrite to OS sep or neither — pick "neither" if Codex tolerates forward slashes on Windows.
