## Findings

### F1-01 — Keyboard handler is a 100-line if-chain with a 19-entry dep array
- **Category**: Maintainability / testability
- **Confidence**: high
- **Diff**: medium
- **Location**: `src/App.tsx:325-429`
- **Today**: One `useEffect` registers a single `keydown` listener whose handler is an `if`-chain over 11 distinct shortcuts. The dep array is 19 entries; almost any state change re-registers the listener. Cycle math (`Ctrl+Tab`) and index math (`Alt+1-9`) sit inline.
- **Proposed**: `useKeyboardShortcuts({ openLauncher, quickLaunch, closeActiveTab, togglePalette, openResumePicker, openSearchPanel, toggleConfigManager, openDevtools, dismissTopmost, cycleTabs, jumpToTab })`. Use a stable `actionsRef` like `useNotifications` so the listener registers once with `[]` deps. Express shortcuts as a data table. Move cycle/index math to `src/lib/tabCycle.ts` for pure-function tests.
- **Side-effects**: Listener stops re-registering; no behaviour change.

### F1-02 — Escape dismissal chain: 8-step nested if-return, no single source of truth
- **Category**: Architecture / correctness
- **Confidence**: high
- **Diff**: medium
- **Location**: `src/App.tsx:383-402`
- **Today**: Hard-coded `if (showX) { ...; return; }` ladder. Doc drift: rule [KB-09 L383] references sidePanel(debug|activity|search) Escape interception not implemented in code.
- **Proposed**: `useDismissalStack` with `register(id, { open, dismiss, intercepts? })` API and a module-level `LAYER_ORDER` array. Bottom-of-stack `id: "terminal"` handles the blur-then-PTY-ESC fallback. Palette registers `intercepts: true`.
- **Side-effects**: Order becomes data; reducer is pure-testable.

### F1-03 — Modal flag soup: 6 booleans/objects with implicit "only-one-at-a-time" coupling
- **Category**: Architecture / state hygiene
- **Confidence**: high
- **Diff**: medium-large
- **Location**: `src/App.tsx:70-90` (decls), `:798-833` (renders)
- **Today**: 2 store-backed (`showLauncher`, `showConfigManager`) + 7 local pieces. No invariant prevents two modals being open simultaneously.
- **Proposed**: Collapse the 7 local pieces into a `useReducer` with discriminated-union `ModalState`: `none | palette | resumePicker | contextViewer | changelog | inspector | tabContextMenu | pruneConfirm`. Reducer enforces mutual exclusion.

### F1-04 — Settled-state feed effect duplicates store subscription
- **Category**: Architecture / clarity
- **Confidence**: medium
- **Diff**: small
- **Location**: `src/App.tsx:99-110, 113-118`
- **Today**: Subscribes to `subagentMap` then re-reads `useSessionStore.getState().subagents` inside the effect — dead read. Two effects (feed + subscribe).
- **Proposed**: `useSettledTabs(): Map<string, SettledKind>` collapses both effects, drops the `_getTrackedSessions` private API call.

### F1-05 — Startup bootstrap mixes 8 concerns in one effect
- **Category**: Cohesion
- **Confidence**: high
- **Diff**: small
- **Location**: `src/App.tsx:121-141`
- **Today**: `loadRuntimeInfo`, `init`, ui-config, past-sessions, prune-recent-dirs, legacy-data migrate, session cleanup, version load+update+CLI checks, weather. 5 inline `.catch(() => {})`.
- **Proposed**: `useStartupBootstrap()` — block on `loadRuntimeInfo → init`, fan-out the rest with `Promise.allSettled`.

### F1-06 — Persistence: 2-second `setTimeout` debounce in App
- **Category**: Locality
- **Confidence**: medium
- **Diff**: small
- **Location**: `src/App.tsx:233-238` + `241-249`
- **Today**: 2000ms timer resets on every `sessions` change. `beforeunload` flush is a separate effect.
- **Proposed**: `useSessionPersistence()` — owns both the debounce and the unload flush, lives next to `useSessionStore`. (Debouncing writes is rate-capping, not caching.)

### F1-11 — Inline prune-confirm dialog hides an arbitrary 8000ms ConPTY-kill workaround
- **Category**: Componentization + CLAUDE.md compliance
- **Confidence**: high
- **Diff**: small
- **Location**: `src/App.tsx:836-872`, specifically `L862` `Promise.race(killPty, setTimeout(8000))`
- **Today**: 8000ms is a hard arbitrary timeout. Per CLAUDE.md "no arbitrary delays" — real design wart.
- **Proposed**: Component split into `PruneDialog.tsx`. **Flag for the user**: deterministic fix is to make `killPty` itself bound its deadline in the Rust backend (R6/R7) so the frontend can simply `await killPty(id)`.
