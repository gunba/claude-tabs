## Findings

### F1 — `observability_enabled()` is a compile-time switch — production has zero observability
- **Category**: design / debuggability
- **Confidence**: high
- **Diff**: M (~80 LOC across mod.rs + 1 frontend hook + settings UI)
- **Location**: mod.rs:19-21
- **Today**: `fn observability_enabled() -> bool { cfg!(debug_assertions) }`. Every gate returns early in release. A release build hitting a real-world Codex/Claude bug logs nothing.
- **Proposed**: Replace compile-time fn with runtime resolver ORing three sources: (1) `cfg!(debug_assertions)` always-on for dev; (2) `CODE_TABS_OBSERVABILITY=1` env var; (3) persisted setting in existing `ui_config`. Use `AtomicBool` flag (config-state mirroring, NOT data caching). Toggle calls back into `runtime.ts` to start/stop bridge. Add `runtimeOverride: bool` field to `ObservabilityInfo`.
- **Side-effects**: F2 (settings/runtime), F9 (ConfigManager) — adds switch. R10 needs new command registered.

### F2 — `append_lines` reopens file on every call — disk thrash
- **Category**: performance / correctness
- **Confidence**: high
- **Diff**: M (~120 LOC: writer actor + Drop on shutdown)
- **Location**: mod.rs:36-48
- **Today**: Every call does `OpenOptions::new().create(true).append(true).open(&path)` → write → `metadata()` → close. Hundreds of opens/min. `metadata()` syscall consumed only by `debugLog.ts:278` for telemetry it doesn't use.
- **Proposed**: `WriterPool` actor — one tokio task per distinct path, mpsc channel, holds single `BufWriter<File>`, flushes on size threshold (~64 KB) or time deadline (~250 ms). On Drop / `RunEvent::Exit`, flushes. `record_backend_event` becomes lock-free for path lookup. Drop `metadata()` syscall.

### F3 — Per-session `notify` watcher — no concurrency bound
- **Category**: resource leak / scalability
- **Confidence**: medium-high
- **Diff**: L (~200 LOC: shared watcher actor)
- **Location**: codex_rollout.rs:139 (spawn), :284 (wait), :323 (tail)
- **Today**: Each `start_codex_rollout` spawns tokio task owning TWO `notify::recommended_watcher` instances. Linux inotify caps: `fs.inotify.max_user_watches` ~8K, `max_user_instances` 128. **Every Codex session watches the same directory** during wait phase — N sessions × 1 directory = N redundant kernel handles for same path.
- **Proposed**: Single shared `RolloutDirWatcher` actor in `CodexRolloutState`. One inotify on today's dir, mpsc broadcast to all waiting sessions. Each `start_codex_rollout` registers `(session_id, spawn_time, oneshot::Sender<PathBuf>)` interest. Per-session **tail** watchers stay 1:1 (each rollout file unique) but bounded by `Semaphore(MAX_CONCURRENT_TAILS=64)`.

### F4 — Silent fall-throughs in event/argument parsing
- **Category**: correctness / debuggability
- **Confidence**: high
- **Diff**: S (~30 LOC + tests)
- **Location**: codex_rollout.rs:392-406 (parse fail), :973-1020 (function-call args fallback)
- **Today**: JSON parse failures already log via `record_backend_event(... "codex.rollout.parse_failed" ...)` — partially addressed. But warn-path drops line bytes — no quarantine, no rate limit. `arguments` fallback at :980-983 (`payload.get("arguments").or_else(|| payload.get("input"))`) IS silent. Note: assignment referenced "L1696" — that line does NOT exist in this 1141-line file.
- **Proposed**: `RolloutParseError` enum (BadJson, MissingType, MissingPayload). Append offending raw line (truncated to 1 KiB) to per-session `codex-rollout-quarantine.jsonl`. Token-bucket rate-limit WARN. For `arguments` fallback, emit `codex.rollout.tool_args_missing` WARN.

### F5 — Prompt-capture dedup key is fragile across CLI feature changes
- **Category**: correctness
- **Confidence**: medium
- **Diff**: S (~25 LOC + 2 tests)
- **Location**: codex_rollout.rs:612-622
- **Today**: Dedup key is `format!("{}\u{1f}{}\u{1f}{}\u{1f}{}", model, base, developer, user)`. Codex `turn_context` may emit fields we don't read (tools list, sandbox, MCP config); those NOT in key. Re-issuing same prompt under different config suppresses tap entry, leaving frontend with stale config attribution. `format!(...)` allocates O(prompt size) per turn_context — 50KB system prompt is hot allocation.
- **Proposed**: Switch dedup key to `u64` SipHash/fxhash of entire `turn_context` payload normalized to canonical JSON. Cheap; captures every field. Decouple dedup from message-extraction.

### F6 — `CodexRolloutState` uses `std::sync::Mutex` from async commands
- **Category**: subtle async safety
- **Confidence**: medium
- **Diff**: S (~10 LOC)
- **Location**: codex_rollout.rs:183, 207-218, 228-233
- **Today**: `watchers` and `claimed_rollouts` are `std::sync::Mutex`. Tauri commands are `async fn`. `lock()` calls in async context risk blocking tokio thread. Prior-handle stop at L213-218 is `tauri::async_runtime::spawn(async move { p.stop().await })` because `prev.stop()` is async — fire-and-forget creates real race window.
- **Proposed**: Switch `watchers` to `tokio::sync::Mutex` (or `DashMap`). Await-while-locked becomes legal; spawn-fire-and-forget collapses to `prev.stop().await` inline. Run `read_dir` portion of `find_unclaimed_rollout` via `tokio::task::spawn_blocking`.

### F7 — `record_backend_event` always emits to Tauri AND writes to disk — no level filter
- **Category**: performance
- **Confidence**: high
- **Diff**: M (~50 LOC)
- **Location**: mod.rs:122-152
- **Today**: When observability is on, EVERY backend event triggers (a) `serde_json::to_string`, (b) `append_lines` (one open/write/stat — F2), (c) `app.emit("observability-entry", entry)`. Frontend listens for every emit. No configurable level threshold — DEBUG pays same cost as ERR. Proxy at `proxy/mod.rs:120-127` records DEBUG `proxy.connection_accepted` for every TCP accept.
- **Proposed**: Static `OBS_MIN_LEVEL: AtomicU8` initialized from env (`CODE_TABS_OBSERVABILITY_LEVEL`, default DEBUG dev / LOG release). `record_backend_event` early-returns when `level < min`. Mirror in `ObservabilityInfo`. Pair with F2 writer pool: serialize once, send same `Value` to (i) writer pool and (ii) Tauri emit. Add 1/N sampler for DEBUG perf-spans.

### F8 — No log rotation — `observability.jsonl` grows unbounded
- **Category**: durability / disk hygiene
- **Confidence**: high
- **Diff**: S (~40 LOC)
- **Location**: mod.rs:23-34, 36-48
- **Today**: `observability_path` returns `<data>/observability/app.jsonl` (global) or `<session>/observability.jsonl` (per-session). Nothing rotates. With F1 (always-on prod), long-running app accumulates indefinitely.
- **Proposed**: In writer-pool actor (F2), check file size on each flush; on threshold (~50 MiB) `std::fs::rename` to `app.jsonl.1`. Keep N=3 rotated files; delete oldest. Expose size + rotation count in `get_observability_info`.
- **Cross-platform**: Rename works on both; on Windows writer must close handle before rename, which buffer-pool actor does naturally.

### F9 — `wait_for_new_rollout` re-walks `read_dir` on every notify event
- **Category**: performance (minor)
- **Confidence**: high
- **Diff**: XS (~15 LOC)
- **Location**: codex_rollout.rs:303-321
- **Today**: Every notify Create/Modify event triggers `claim_unclaimed_rollout` → `std::fs::read_dir(dir)` → walk every entry. On busy day with N rollouts, every event is O(N) syscalls.
- **Proposed**: Notify event payload contains affected paths (`ev.paths`). Pass `Vec<PathBuf>` through channel; only inspect those.

### F10 — `record_backend_event` allocates twice per event for serialization
- **Category**: micro-performance
- **Confidence**: high
- **Diff**: XS (~5 LOC, depends on F2)
- **Location**: mod.rs:148-151
- **Today**: `serde_json::to_string(&entry)` then `line + "\n"` allocates twice. `app.emit` re-serializes again across IPC.
- **Proposed**: With F2 in place, write to `BufWriter<File>` via `serde_json::to_writer` (single alloc-free pass).

## Cross-cutting concerns

- **F5 tap pipeline contract**: `tap-entry-{sid}` events at codex_rollout.rs:430-433 consumed by `useTapPipeline.ts:138`. None of proposals change JSON shape. F4 adds new WARN records on different event name (`codex.rollout.tool_args_missing`); F5 may emit `cat=system-prompt` more often, unchanged shape.
- **R9 (tap_server)**: `tap_server.rs:148` uses identical event-name pattern + `record_backend_event`. F2 writer pool is shared infra — R9 benefits transparently.

- **F2 settings/runtime + F9 ConfigManager**: F1 needs UI toggle in Settings → Debug; persisted in `ui_config`; new `set_observability_enabled` command registered in `lib.rs:188` (R10).
