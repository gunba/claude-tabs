## Header Activity Visualizer
Ambient soundwave-and-mascots scene that fills the gap between the tab strip and the right-side action buttons

- [HA-01 L12] HeaderActivityViz paints the slack between .tab-bar-scroll and the right-hand buttons (.tab-resume / .tab-config / .tab-add) into a single <canvas> as an ASCII terminal scene. Cell grid: CELL_W=7px, CELL_H=12px, FONT_PX=11; cols = floor(width/CELL_W), rows = floor(height/CELL_H). Backing store is dpr*cssSize with ctx.scale(dpr,dpr) so text stays sharp on HiDPI; ResizeObserver watches the wrapper and a MutationObserver on document.documentElement style attribute re-reads CSS vars (bgSurface, textMuted/textSecondary, cliClaude, cliCodex, error, fontMono) when applyTheme mutates root inline style. All paint runs inside one rAF tick — no per-bar refs, no per-mascot DOM, no bubble/weather divs. Wave geometry is in cell space, not pixel space, so the visual is identical at every monitor width: WAVE_PERIOD_CELLS=14, WAVE_SPEED_CELLS_PER_SEC=4, WAVE_AMPLITUDE_ROWS=1.3; per column c at time t seconds, phase=(c/14 - 4t/14)*2π and crestRow = (rows-2) - sin(phase)*1.3*intensity. Wave is drawn for cols BEACH_COLS..cols-1; each column paints '^' at the local crest minimum and '~' for the body to the bottom (textSecondary 0.85 / textMuted 0.5 alpha). Beach is fixed at BEACH_COLS=8 cells on the left, rows beachTopRow..rows-1, with a deterministic glyph pattern from BEACH_GLYPHS (.,;:') indexed by (col*31+row*7)%5 — light glyphs render in #c8a868, dark in #a07c44. Six weather scenes drawn directly on the canvas: 'clear' paints a 3x3 ASCII sun (\\|/ -O- /|\\) anchored at (cols-4, 0) AFTER the wave so the wave can't overpaint it (drawSky's 'clear' branch is intentionally a no-op); 'clouds' drifts 3 '___' blocks at 1.4–2.4 cells/sec; 'rain' draws per-column slanted apostrophes at ~1.4 cycles/sec; 'snow' floats up to 14 '*'/'.' flakes with sin-sway and respawn at the top; 'fog' dims a band of '~' across alternate columns just above the wave baseline; 'storm' is rain plus a single-frame rgba(220,230,255) flash overlay every STORM_PERIOD_S=7 seconds (STORM_FLASH_S=0.1s decay). Cloud and snowflake arrays are reset on weather scene change. Agents tracked in cell coordinates in a persistent slotsMapRef Map<id, Slot>: colF (current column float), dir (1|-1; surfers bounce between BEACH_COLS and cols-1 instead of wrapping), homeCol/homeRow (deterministic beach rest from hash32(id)), speed (5–9 cells/sec), jitterSeed (bubble x-jitter), diveT, bubbleAccumMs. Glyphs: '&' Claude main / '@' Codex main / 'o' subagent (either CLI) / 'x' errored or interrupted. Color: var(--cli-claude) #d4744a for Claude, var(--cli-codex) #39c5cf for Codex, var(--error) for errored; alpha 0.9 active / 0.45–0.6 idle / scaled 0.7 for completed subagents / faded by diveT for errored. Active agents render at floor(crest)-1; idle at beachTopRow+homeRow; errored at floor(crest)+floor(diveT*2) and emit '°' bubbles starting at agentRow-1 every BUBBLE_INTERVAL_MS=320ms, lifetime BUBBLE_LIFE_MS=1200ms, drifting up at 2 rows/sec. Activity intensity samples session.metadata.toolCount deltas every 500ms via useSessionStore.getState() (no subscription, no per-frame re-render); burst = 1-exp(-delta/3), EMA intensityRef.current * 0.7 + (0.15 + 0.85*burst) * 0.3 drives WAVE_AMPLITUDE_ROWS scaling so the wave breathes with activity. lastCounts seeds new sessions with their current count so historical activity isn't replayed as a burst. Component is aria-hidden, pointer-events:none — pure decoration. CSS shrinks to ~20 lines: .header-activity-viz (flex:1 1 0; min-width:0; overflow:hidden; pointer-events:none; border-left/right:1px solid var(--border-subtle)) wrapping a .header-activity-viz-canvas (display:block; width:100%; height:100%). Pure helpers (hash32, makeSlotInit, withAlpha, beachGlyph, BEACH_GLYPHS, clampCol) are exported and unit-tested in __tests__/HeaderActivityViz.test.ts.
