## Activity Panel

- [AP-04 L20] ActivityPanel shows a floating/sticky mascot that travels to the file currently being accessed by the main agent. The mascot is provider-aware (cli prop drives MASCOT_SRC: claude-mascot.png or codex-mascot.png). Animations: reading/writing/searching rock, moving hops, idle subtly bobs. Mascot animation is suppressed on tab switch via StickyMascot.tabId. Subagent files show inline AgentTypeIcon indicators at their last-touched files while active, and completed subagents remain dimmed at their last-touched files. Subagent indicators share .agent-mascot-img animation/overlay behavior; .agent-mascot-icon defaults to var(--cli-claude) and .agent-mascot-cli-codex overrides it to var(--cli-codex). Tree indentation uses INDENT_STEP=16px.
- [AP-05 L21] ActivityPanel has two view modes toggled by a button group: 'Response' (files touched since lastUserMessageAt, filtered per-file by timestamp) and 'Session' (all visited paths via allFiles/visitedPaths). File state communicated through filename color only: created=green (--success), modified=yellow (--warning), deleted=red+strikethrough (--error), read=muted (--text-secondary) via file-tree-kind-* CSS classes. No letter badges.
- [AP-06 L318] Search activity is recorded for searched folders but rendered as low-signal neutral activity, not provider blue/cyan. Write-side precedence in addFileActivity() still prevents subagent searched entries from overriding main-agent entries for the same path. Render-side in ActivityPanel.tsx still applies file-tree-searched only when node.activity.kind==='searched' and the searching agent is main/unknown or still active, and never on the workspace root; the CSS maps searched names/icons to muted text so workspace/provider colors remain legible. removeSubagent() calls clearAgentSearchActivity() to drop stale subagent folder-search entries.
- [AP-07 L385] ActivityPanel renders a header band (.activity-panel-header) above the scrollable tree: a 32px AgentMascot with hideOverlay=true sitting on a pulsing radial-gradient clay glow (panel-header-glow-pulse keyframes, 4s ease-in-out), followed by a two-line narration. Line 1 is verb + kind-tinted target chip (.panel-header-target with .file-tree-kind-{created|modified|deleted|read|searched} background tints) + optional 'Ns ago' detail in idle. Line 2 is the workspace-relative path in mono (.file-tree-path-mono → var(--font-mono)). Verbs derive from currentToolName via toolToVerb(): Read→Reading, Edit/NotebookEdit→Editing, Write→Writing, Grep/Glob/LSP→Searching, Bash→Running. Idle fallback uses lastMainAgentFile.kind for the chip color and relativeTime() for the detail. Bash never reaches the chip path because Bash is excluded from FILE_TOOLS in primaryActive resolution; falls through to idle narration. workspaceRelative() strips the canonicalized workspace prefix when present.
- [AP-08 L492] ActivityPanel renders a translucent breadcrumb of recent main-agent file positions inside .activity-tree-container. recentMainAgentPaths walks activity.turns newest-first to collect the last FOOTPRINT_LIMIT+1 (=6) distinct main-agent (agentId=null) paths. The footprint-positioning effect filters out the current floating-mascot target, then renders up to 5 .activity-footprint dots positioned absolutely at the matching row's left margin (data-path query, depth-driven left). Opacity steps 0.42 → 0.08 by age via Math.max(0.08, 0.42 - age*0.08); the age counter increments on collapsed/missing rows so visible footprints retain temporal ordering. Cleared on tab switch alongside expandedFiles via the activeTabId useEffect, and re-derived in the same effect pattern as the floating mascot when rows/depthByPath change.
- [AP-09 L779] ActivityPanel changes the file-row click handler from invoke('shell_open') to toggling local expandedFiles Set state; shell_open moves to a dedicated .file-tree-open-btn (IconExternalLink) on the right edge with stopPropagation. Expanded rows render <DiffCard> directly underneath the row, indented at depth*INDENT_STEP+24px. buildUnifiedDiff splits oldString/newString on newline, walks longest common prefix and suffix, emits up to 2 lines of context before/after the diff range, marks removed lines red and added lines green via .file-tree-diff-{context|added|removed} with -/+ gutter glyphs, and caps total at 24 lines. Write activities render the first 24 lines of toolInputData.content as additions.
